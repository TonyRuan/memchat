# 记忆提取触发策略偏好

## 类型

preference

## 状态

active

## 可信度

90

## 背景

用户确认 MemoryChat 的记忆整理不应在每轮普通对话都调用模型，而应该后台异步运行，以降低 API 成本和减少对聊天体验的影响。

## 当前结论

- 用户明确说“记住”“以后记得”“加入记忆”等指令时，必须立即后台触发记忆提取。
- 普通对话默认累计到批量阈值后再后台整理。
- 用户离开当前会话时，如果仍有未整理消息，应后台触发一次整理。
- 手动整理按钮仍然立即执行，保留 PRD 验收路径。

## 适用范围

适用于 MemoryChat 的自动记忆提取、手动记忆整理、ADB 调试链路和端到端记忆验证。
