# 记忆提取必须做端到端验证

## 类型

lesson

## 状态

active

## 可信度

95

## 背景

用户在真实使用 MemoryChat 时反馈：记忆提取链路存在明显问题，包括主动提取按钮按了没有效果、记忆中心看不到结果、界面出现“学会了”后后续对话没有体现该记忆，以及后台自动提取也没有明显生效。

## 当前结论

记忆功能不能只靠单元测试或模型调用成功来判断完成；必须端到端验证“触发提取 -> 写入数据库 -> 记忆中心展示 -> 后续召回影响回复”的完整闭环。v1.0.41 的最终 ADB 验证证明：使用 Base64 debug 输入后，显式记忆可自动写入 `memories`，后续提问能召回并回答保存内容。v1.0.43 的后台批量提取还暴露一个并发边界：只用 `ChatViewModel.memoryExtractionJob` 做防重入无法覆盖快速退出、重进、再退出导致的新 ViewModel 实例；后台提取必须有 Application 级按 conversationId 的 in-flight guard。

## 适用边界

- 适用于记忆提取、保存、去重、禁用、删除、召回和记忆中心 UI。
- 适用于主动提取按钮、后台自动提取和真实模型 smoke。
- 需要用模拟器/真机验证时，优先派子代理执行高输出日志采集，主线程整合结论。

## 后续动作

- 先复现并定位失败发生在 UI 触发、提取器、解析、保存、过滤、展示还是召回环节。
- 修复后用数据库、logcat、UI 和后续对话结果共同验收。
